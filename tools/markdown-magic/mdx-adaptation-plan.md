# Plan: MDX-aware Content Adaptation in markdown-magic

## Context

When `INCLUDE` (or any transform) embeds content from a `.md` source file into an `.mdx` target file, HTML comments (`<!-- -->`) in the embedded content are invalid MDX. MDX v2+ treats HTML comments as raw HTML nodes, which can cause parse errors or unexpected rendering. Two concrete examples appeared when embedding `packages/dds/sequence/README.md` into `sequences.mdx`:

- `<!-- When talking about positions... -->` — a commented-out paragraph  
- `<!-- This line ends the content... -->` — an editorial marker

The fix is to parse the content returned by every transform using remark, detect HTML comment nodes, and convert them to MDX-compatible JSX expression comments (`{/* */}`) when the target file is `.mdx`. This should be done in the processor as a post-processing step so it applies uniformly to all transforms.

## Approach

**Single post-processing step in the processor.** After any transform returns its content string, if the target file is `.mdx`, run the content through `adaptMarkdownForMdx()`. This function:

1. Parses the content string with `unified().use(remarkParse)` — the same parser already used in `processFile` for the target file itself
2. Visits all `html` AST nodes (which represent block-level HTML in Markdown, including `<!-- -->` comments)
3. For each HTML comment node, records its source offset range and the comment body text
4. Applies string-level replacements in reverse document order (to preserve earlier offsets), converting `<!-- body -->` → `{/* body */}`
5. Returns the transformed string

This reuses the same pattern already used by `processFile` for pragma detection: parse to AST → collect positions → splice at offsets. It does **not** require `remark-stringify` (not a current dependency) and makes no changes outside HTML comment nodes.

`html` nodes produced by remark-parse only represent block-level HTML. HTML inside code fences lives inside `code` nodes and is never visited, so code examples are safe.

## Files to Modify

### 1. `src/utilities.ts` — add `adaptMarkdownForMdx`

Add imports for `unified`, `remarkParse`, and `visit` (already in `package.json`; already imported in `processor.ts`).

Add and export:

```typescript
/**
 * Parses the provided Markdown content string and converts any HTML comment nodes
 * (<!-- ... -->) to MDX-compatible JSX expression comments ({/* ... *\/}).
 *
 * Safe for content with code fences — remark-parse represents fenced code as `code`
 * nodes, not `html` nodes, so comment-like text inside code blocks is never modified.
 */
export function adaptMarkdownForMdx(content: string): string {
    const tree = unified().use(remarkParse).parse(content);

    const replacements: Array<{ start: number; end: number; body: string }> = [];
    visit(tree, "html", (node: any) => {
        const match = (node.value as string).match(/^<!--([\s\S]*?)-->$/);
        if (match) {
            replacements.push({
                start: node.position.start.offset,
                end: node.position.end.offset,
                body: match[1],
            });
        }
    });

    let result = content;
    for (const { start, end, body } of [...replacements].reverse()) {
        result = result.slice(0, start) + `{/*${body}*/}` + result.slice(end);
    }
    return result;
}
```

### 2. `src/processor.ts` — call `adaptMarkdownForMdx` in `processFile`

Import `adaptMarkdownForMdx` from `"./utilities.js"`.

After the transform result is awaited and the undefined/null guard runs (currently ~line 237), add:

```typescript
if (isMdx) {

```
